# Exercício 02 – Recife Tech Story

## Objetivo

Praticar criação de arquivos e organização em pastas, escrevendo um pequeno texto em Markdown sobre tecnologia, inovação ou cultura digital em Recife.

---

## Tarefa

Crie um arquivo Markdown com uma pequena história, relato ou descrição relacionada ao ecossistema tecnológico do Recife.

### Passo a passo

1. Crie uma branch específica:

   ```bash
   git checkout -b ex02-seu-nome
   ```

2. Crie um arquivo dentro desta pasta usando o nome:

   ```bash
   historia-<seu-usuario-github>.md
   ```

3. Dentro do arquivo, use este template:

   ```markdown
   # Título da sua história

   ## Sobre o que é esta história?
   Explique em 2–3 frases o foco (ex.: um evento, um lugar, um momento, uma experiência).

   ## Relato
   Escreva um pequeno texto (5–10 frases) contando uma história ou descrição:
   - Pode ser real ou fictícia  
   - Deve ter relação com tecnologia, inovação ou cultura digital em Recife

   ## Por que isso é relevante para alguém da área de computação?
   Conecte sua história com a vida de um estudante ou profissional de tecnologia.
   ```

4. Ao terminar, faça commit e push:

   ```bash
   git add exercicios/02-recife-tech-story/historia-<seu-usuario>.md
   git commit -m "adiciona historia de <seu-usuario>"
   git push origin ex02-seu-nome
   ```

5. Abra um Pull Request para o repositório do professor.

---

## Critérios de avaliação

- Branch criada corretamente1
- Nome do arquivo seguindo o padrão
- Template preenchido
- História coerente e relacionada a tecnologia/inovação em Recife
- Commit claro
- PR aberto corretamente
